开发环境：使用Ruby工具链统一开发环境

  1. 电脑mac环境安装统一的 ruby 环境和 Xcode 代码编译环境

    • 在执行 bundle exec fastlane test 之前，需要使用xcode-select -s /Applications/Xcode12.2.app/Contents/Developer 来固定对应的Xcode版本

    • Xcode 的对应版本需要到苹果的开发者网站搜索并下载

    • 然后确定安装好了homebrew并且版本较新，一般mac自带而且版本不会太大问题。

    • 安装和卸载命令分别是（可能会失效去官网复制使用最新的命令）：

  2. 安装ruby版本管理工具 rbenv

    一旦安装 rbenv 完毕，我们需要把以下的设置信息放到你的 Shell 配置文件里面，例如 ~/.bash_profile 或者 ~/.zshrc 等文件，这样能保证每次打开终端的时候都会初始化 rbenv。

    如果发现本地能安装的版本没有3.0.0，就需要更新一下brew brew update && brew upgrade ruby-build, brew update 出现问题，需要git 更新仓库，替换源并且更新内容。

  3. 安装ruby需要用到的依赖库管理工具 RubyGems 和 Bundler

    gem install bundler 之后 执行 bundle init 就可以生成一个 Gemfile 文件，像 CocoaPods 和 fastlane 等依赖包，我们就可以添加到这个文件里面。然后执行 bundle install 来安装各个 Gem。 Bundler 会自动生成一个 Gemfile.lock 文件来锁定所安装的 Gem 的版本。

  4. Bundler 可以有效管理 CocoaPods 和 fastlane

  5. 最后在工程里创建一个 shell 脚本，新项目或者新电脑只要运行一下这个脚本就可以构建出一套iOS的开发环境。
